Replace the stub content in agents/topgun-installer.md with full installation logic. Keep the existing frontmatter (name, description, model, color, tools).

The agent receives context from the orchestrator including: secured copy path (~/.topgun/secured/{sha}/SKILL.md), skill name, source registry, install_url (from found-skills data), and comparison/audit metadata.

Implement the following flow:

## Step 1: Read Install Context

Read state to get the secured skill path and metadata:
```bash
node "$CLAUDE_PLUGIN_ROOT/bin/topgun-tools.cjs" state-read
```

Extract: secured_path, skill_name, install_url, source_registry from state. Also read the audit-{hash}.json and comparison-{hash}.json to get full metadata.

## Step 2: Attempt /plugin install

Try the primary install path:

```bash
/plugin install {install_url}
```

If install_url is not available or is a local secured copy, use the local path:
```bash
/plugin install {secured_path}
```

Capture the output. If the command exits non-zero or outputs an error, set `plugin_install_failed = true` and proceed to Step 4 (fallback — handled by Plan 05-03).

## Step 3: Post-Install Verification (REQ-20)

Run two verification checks:

**Check 1: installed_plugins.json verification**
```bash
cat ~/.claude/installed_plugins.json 2>/dev/null
```

Search the JSON for an entry matching the skill name or install path. 

If entry is MISSING (GitHub issue #12457 mitigation): write the entry manually. Read the current installed_plugins.json, append a new entry with the skill's name, path, and install timestamp, then write the file back:

```bash
node -e "
const fs = require('fs');
const path = require('path');
const pluginsPath = path.join(process.env.HOME, '.claude', 'installed_plugins.json');
let plugins = [];
try { plugins = JSON.parse(fs.readFileSync(pluginsPath, 'utf8')); } catch(e) { plugins = []; }
plugins.push({
  name: '{skill_name}',
  path: '{install_path}',
  installed_at: new Date().toISOString(),
  source: 'topgun'
});
fs.writeFileSync(pluginsPath, JSON.stringify(plugins, null, 2));
console.log('Entry written manually (#12457 mitigation)');
"
```

**Check 2: Test invocation**

Attempt to invoke the installed skill with a no-op test:
- Use the Skill tool or Task tool to call the installed skill with a minimal test prompt
- If the skill responds (any non-error response), mark test_invoke = passed
- If the skill errors or is not found, mark test_invoke = failed

**Decision matrix:**
- installed_plugins OK + test_invoke OK → installation verified, proceed to ## INSTALL COMPLETE
- installed_plugins OK + test_invoke FAILED → warn but continue (skill registered, may need restart)
- installed_plugins FIXED + test_invoke OK → installation verified (manual fix worked)
- installed_plugins FAILED + test_invoke FAILED → both checks failed. Output error message: "Post-install verification failed. Skill not found in installed_plugins.json and test invocation failed." Set `install_verified = false` in state. Signal to orchestrator that local-copy fallback is needed.

Write verification results to state:
```bash
node "$CLAUDE_PLUGIN_ROOT/bin/topgun-tools.cjs" state-write install_method "plugin"
node "$CLAUDE_PLUGIN_ROOT/bin/topgun-tools.cjs" state-write install_verified "{true|false}"
node "$CLAUDE_PLUGIN_ROOT/bin/topgun-tools.cjs" state-write plugins_json_status "{found|manual_fix|missing}"
node "$CLAUDE_PLUGIN_ROOT/bin/topgun-tools.cjs" state-write test_invoke_status "{passed|failed}"
```

## Step 4: Completion or Fallback Signal

If install verified:
```
## INSTALL COMPLETE
```

If not verified (both checks failed) OR plugin install failed:
```
## INSTALL FAILED — FALLBACK NEEDED
```

The orchestrator (Plan 05-03) handles the fallback path.
