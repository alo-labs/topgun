  <action>
Replace the current Step 6 stub ("Not implemented in Phase 1") in skills/topgun/SKILL.md with full approval gate logic:

1. Read the audit results from ~/.topgun/audit-{hash}.json (written by SecureSkills in Step 5). Extract: skill name, source registry, composite score (capability/security/popularity/recency), Sentinel summary (pass count, finding count), allowed-tools list, and the secured copy path.

2. Read the comparison results from ~/.topgun/comparison-{hash}.json to get the winner's metadata (name, source_registry, scores).

3. Present the audit manifest to the user in this exact format:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 TOPGUN ► APPROVAL REQUIRED
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 Skill:       {name} ({source_registry})
 Score:       capability={X} / security={X} / popularity={X} / recency={X}
 Sentinel:    {pass_count} passes, {finding_count} findings resolved
 Allowed-tools: {comma-separated list}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

4. BEFORE the approval prompt, check the allowed-tools list. If it contains "Bash", "Computer", or a wildcard ("*"), display an explicit permission warning (per REQ-18):

```
⚠ WARNING: This skill requests elevated permissions:
  - {list each dangerous tool}
  These tools allow the skill to execute arbitrary commands on your system.
  Review the allowed-tools list carefully before approving.
```

5. Ask the user: "Do you approve installation of this skill? (yes/no)"

6. If user says "yes" (or "y", case-insensitive):
   - Update state:
     ```bash
     node "$CLAUDE_PLUGIN_ROOT/bin/topgun-tools.cjs" state-write last_completed_stage approve
     node "$CLAUDE_PLUGIN_ROOT/bin/topgun-tools.cjs" state-write approval "approved"
     node "$CLAUDE_PLUGIN_ROOT/bin/topgun-tools.cjs" state-write approved_at "$(date -u +%Y-%m-%dT%H:%M:%SZ)"
     ```
   - Proceed to Step 7 (InstallSkills)

7. If user says "no" (or "n", case-insensitive):
   - Update state:
     ```bash
     node "$CLAUDE_PLUGIN_ROOT/bin/topgun-tools.cjs" state-write last_completed_stage approve
     node "$CLAUDE_PLUGIN_ROOT/bin/topgun-tools.cjs" state-write approval "rejected"
     node "$CLAUDE_PLUGIN_ROOT/bin/topgun-tools.cjs" state-write current_stage complete
     ```
   - Output: "Installation rejected by user. Pipeline complete — no skill installed."
   - STOP. Do NOT proceed to Step 7.

8. Also update the resume check in Step 2 to handle the "secure" case correctly: when resuming from `secure`, go to Step 6 (approval gate), not skip to Step 7.
  </action>

  </verify>
